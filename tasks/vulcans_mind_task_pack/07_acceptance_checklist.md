# 07. Acceptance checklist — Vulcan’s Mind presentation polish

## 1. Visual design

- [ ] Чат має нейтральну сіру/асфальтову тему або акуратну світлу graphite-тему.
- [ ] Інтерфейс не виглядає як пісочна документаційна картка.
- [ ] Header компактний.
- [ ] Trash icon замінено на reset/history icon.
- [ ] Close icon працює.
- [ ] User і assistant messages візуально відрізняються.
- [ ] Текст читається комфортно на мобільному.
- [ ] Акцентний колір використовується помірно.

## 2. Input bar

- [ ] Кнопка «Надіслати» замінена на іконку.
- [ ] Немає overlay проблем на iPhone Safari.
- [ ] Постійний лічильник `0/1500` прибраний.
- [ ] Індикатор ліміту зʼявляється тільки near-limit або при перевищенні.
- [ ] Send button disabled, якщо поле порожнє.
- [ ] Send button disabled або loading під час генерації відповіді.
- [ ] Input має font-size ≥ 16px.

## 3. Welcome / empty state

- [ ] З welcome message прибрана фраза «Спираюсь на обмежену базу публічних джерел.»
- [ ] Є короткий disclaimer: «Не є медичним сервісом і не замінює професійну консультацію.»
- [ ] Показуються тільки 3 випадкові приклади питань.
- [ ] Приклади питань зникають після першого запиту користувача.
- [ ] Після reset/new chat приклади можуть зʼявитися знову.

## 4. Sources UI

- [ ] Блок «ДЖЕРЕЛА» не показується після відповіді.
- [ ] Великі картки джерел не відображаються.
- [ ] Project notes не показуються як broken links.
- [ ] Відсутність sources UI не ламає backend response parsing.

## 5. Mobile Safari

- [ ] При focus на input Safari не робить zoom-in.
- [ ] Клавіатура не перекриває input.
- [ ] Остання відповідь не перекривається input bar.
- [ ] Після 5+ повідомлень можна проскролити вгору і прочитати попередні відповіді.
- [ ] Після відповіді чат не блокує scroll.
- [ ] Auto-scroll не заважає, якщо користувач вручну читає старі повідомлення.
- [ ] Layout не стрибає при focus/blur input.

## 6. Conversation behavior

- [ ] На питання «Над якими проєктами працює Вулкан?» бот відповідає про «Люстерко» і Vulcan’s Mind.
- [ ] Якщо включено в базу, бот може згадати «Іменуй мене» як ширший концепт, але не як готовий продукт.
- [ ] Out-of-scope fallback не звучить як механічна відмова.
- [ ] Бот пропонує корисні суміжні теми, якщо питання поза базою.
- [ ] Бот не вигадує факти про Володимира, партнерства, фінансування або статуси проєктів.
- [ ] Бот не ставить діагнози і не призначає лікування.
- [ ] Бот може пояснювати загальні принципи психометрики, скринінгу, RAG, AI, ментального здоров’я.

## 7. Prompt / RAG

- [ ] Додано або оновлено canonical document `vulcan_profile_and_projects.md`.
- [ ] Питання про Володимира та його проєкти не потрапляють у out-of-scope fallback.
- [ ] System/behavior prompt оновлено.
- [ ] Відповіді стали менш сухими й менш документаційними.
- [ ] Модель винесена в env vars, якщо це не потребує великого рефакторингу.
- [ ] Додано seed list для synthetic QA / golden answers або створено задачу на його додавання.

## 8. Typing / fake streaming

- [ ] Після відправки повідомлення показується typing indicator.
- [ ] Не використовується текст «Шукаю в базі джерел…».
- [ ] Відповідь зʼявляється поступово через fake streaming/typewriter.
- [ ] Typewriter не ламає markdown або після завершення коректно рендерить markdown.
- [ ] Reset/new chat зупиняє typing/typewriter.
- [ ] Error state прибирає typing indicator.

## 9. Regression tests

Перевірити вручну такі сценарії:

- [ ] Відкрити чат на iPhone Safari.
- [ ] Прочитати welcome.
- [ ] Натиснути один із 3 прикладів питань.
- [ ] Перевірити, що приклади зникли після першого повідомлення.
- [ ] Поставити питання: `Над якими проєктами працює Вулкан?`
- [ ] Поставити питання: `Що таке Люстерко?`
- [ ] Поставити питання: `Як AI можна використати в охороні ментального здоров’я?`
- [ ] Поставити out-of-scope питання.
- [ ] Перевірити reset/new chat.
- [ ] Відправити 5–7 повідомлень поспіль.
- [ ] Спробувати скролити вгору після довгої відповіді.
- [ ] Перевірити input focus і клавіатуру.
- [ ] Перевірити desktop viewport.
- [ ] Перевірити Android Chrome, якщо доступний.
