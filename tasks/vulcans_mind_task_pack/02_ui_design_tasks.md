# 02. UI design tasks — Vulcan’s Mind

## Мета

Перевести чат-бот із поточної пісочно-помаранчевої, дещо громіздкої стилістики у чистий нейтральний chat UI, натхненний сучасними AI-чатами, але без прямого копіювання ChatGPT.

## P0.1 — Загальна тема

### Завдання

Оновити дизайн modal/chat widget у нейтральну сіру/асфальтову тему.

### Вимоги

- Фон чату: graphite/asphalt або світлий graphite, залежно від того, що краще вписується у сайт.
- Header компактний, без зайвого декоративного простору.
- Повідомлення мають бути головним візуальним елементом.
- Відповідь асистента має читатися як текст, а не як картка документації.
- User message і assistant message мають візуально відрізнятися, але без агресивного контрасту.
- Акцентний колір можна залишити помаранчевим, але тільки для action states: send button, focus ring, hover/active.

### Не робити

- Не копіювати ChatGPT UI піксель-в-піксель.
- Не використовувати великі декоративні рамки навколо кожного елемента.
- Не залишати великий пісочний блок sources.
- Не робити кнопку «Надіслати» текстовою.

## P0.2 — Header

### Поточна проблема

Header працює, але trash icon створює неправильний сенс: це виглядає як «видалити», а не як «скинути/почати нову розмову».

### Завдання

- Залишити назву `Vulcan's Mind`.
- Замінити trash icon на reset/history icon.
- Закрити modal через `X` залишити.

### Рекомендована іконка

Перевага: `rotate-ccw`.

Альтернативи:

- `history`;
- `clock`.

### Tooltip / aria-label

Для reset icon:

```text
Нова розмова
```

або

```text
Скинути чат
```

Для close icon:

```text
Закрити чат
```

## P0.3 — Message bubbles

### Завдання

Оновити вигляд повідомлень.

### Вимоги

Assistant message:

- ширина: до 100% контейнера, але з комфортними відступами;
- фон: surface/elevated surface;
- border або subtle shadow допустимі, але дуже делікатні;
- line-height: 1.45–1.6;
- font-size на мобільному не менше 16px.

User message:

- вирівняне праворуч;
- не надто яскраве;
- фон трохи контрастніший за assistant bubble;
- не займати 100% ширини без потреби.

### Acceptance criteria

- На iPhone повідомлення читаються без zoom.
- Відповіді не виглядають як великі інформаційні плашки.
- User і assistant візуально відрізняються.

## P0.4 — Input bar

### Поточна проблема

- Текстова кнопка «Надіслати» займає забагато місця.
- Лічильник `0/1500` створює шум.
- Є ризик overlay проблем на вузьких екранах.

### Завдання

Переробити нижній input bar.

### Вимоги

- Textarea/input з font-size ≥ 16px.
- Send button — компактна іконка, не текст.
- Постійний лічильник символів прибрати.
- Якщо користувач наближається до ліміту, показати компактний warning.

### Рекомендований вигляд

- Поле вводу займає всю доступну ширину.
- Кнопка відправки кругла або rounded-square, справа.
- Кнопка disabled, якщо input порожній або йде генерація відповіді.
- При focus — делікатний accent ring.

### Іконка send

Варіанти:

- `arrow-up` у колі;
- `send`;
- `paper-plane`.

Перевага: `arrow-up`, бо це ближче до сучасного chat pattern.

## P0.5 — Character limit behavior

### Завдання

Замість постійного `0/1500` додати contextual indicator.

### Логіка

- До 1200 символів: нічого не показувати.
- 1200–1500: показати маленький muted counter, наприклад `1280/1500`.
- >1500: показати error state і заблокувати send або обрізати відповідно до існуючої логіки.

### Текст помилки

```text
Запит задовгий. Скороти його до 1500 символів.
```

## P1 — Polishing states

### Loading state

Замість «шукаю в базі джерел» показувати typing indicator:

```text
Vulcan’s Mind думає…
```

або тільки animated dots без тексту.

### Error state

Замість технічного повідомлення:

```text
Не вдалося отримати відповідь. Спробуй ще раз або переформулюй питання.
```

### Empty state

Welcome + 3 random question chips.
